처음 블로그 설치, 설정을 고려하는 분들께 도움이 되고자 THTYLE 블로그 설정, 설치 정보를 정리해본다.

블로그 플랫폼

THTYLE 블로그는 Wordpress.org 설치형 플랫폼을 사용한다(Wordpress.com은 서비스형).

 

설치형

사용자가 직접 본인의 호스팅, 도메인을 갖고 그 곳에 블로그 플랫폼을 설치하여 테마, 플러그인 등을 설정한 후 사용하는 형태. 기본 플랫폼외 모든 부분을 커스터마이징 할 수 있어 개인화된 블로그를 가질 수 있는 장점이 있지만, 호스팅, 도메인 확보를 위해서는 일정 사용료를 내야하고 기본적인 웹 관련 지식없이(전문적인 지식까진 필요없지만) 설치, 운영하기에 어려운 단점이 있음. 대표적인 설치형 블로그 플랫폼으로 Wordpress, Textcube(구 테터툴즈), Movabletype이 있다.

 

서비스형

웹 서비스 업체에 가입하면 사용자는 별도의 설정없이 계정, 블로그 플랫폼, 저장공간, 테마, 플러그 인 등을 사용할 수 있다. 기본 서비스를 무료로 이용할 수 있고 쉽게 사용할 수 있는 장점이 있음. 반면에 단점으로는 기능 커스터마이징, 테마 수정, 개인 도메인 사용 등 고급 개인화가 불가능한 것. 대표적인 서비스형 블로그는 Wordress.com, Blogger.com, Tistory.com, 네이버 블로그, 다음 블로그 등이 있다.

 

 

사용 중인 호스팅, 도메인 서비스

호스팅 업체

BYUS.NET (비누넷, 바이어스닷넷) 2002년 개인 홈페이지부터 현재 블로그까지 BYUS.NET에서 사용 중이다. 개인 사용자에 적합한 호스팅 비용과 저장 공간을 제공한다. 관리자가 사용자들의 질문에 빠르게 답변해주고 대응이 빠른 것도 장점이다. 하지만, 서버의 기본 환경이 노후화되어 최신 서비스 사용이 어려운 단점도 갖고 있다.

 

호스팅 기본 서비스 사항

  • Linux Redhat 7.2 (Enigma) / Kernel 2.4.26 #1
  • Apache 1.3.41 / PHP 4.4.1 (Zend Optimizer 2.5.10) / MySQL 3.23.58/4.0.22/5.0.18(utf8): 타 서비스에 비해 버전이 낮은 편임
  • CGI / Perl_5.6.1 (/usr/bin/perl) / Perl DBI/DBD (DBI-1.48,DBD-mysql-3.0002)
  • Python_2.3.3 (/usr/local/bin/python) / MySQL-python-0.9.2
  • Telnet / OpenSSH 1,2 / FTP
  • POP3 / SMTP / IMAP / WEB-MAIL (Sendmail-/usr/sbin/sendmail)
  • GD 2.0.14 / netpbm / ImageMagicK / Jhead (Image tool)

 

도메인

THTYLE.COM 도메인은 닷네임코리아를 통해 사용하고 있다. 도메인 서비스의 상세는 잘 모르지만 가격대비 사용환경이 괜찮은 편인 것 같다.

 

 

Wordpress 플러그인, 테마

플러그인

설치형 Wordpress의 기본 설정 기능들은 서비스형 Wordpress.com에 비해 거의 없다고 보면 된다. 부가기능들을 수행해줄 수 있는 플러그인을 일일이 찾아 설치를 해야하는 번거로움은 있지만 그만큼 자유도가 높아서 원하는 기능으로 설정할 수 있는 장점이 있다. 현재 블로그에서 사용하고 있는 플러그인들을 정리했다.

  • All in One SEO Pack: 블로그 포스트에 메타정보를 쉽게 넣어 SEO(Search Engine Optimization)을 통해 구글, 네이버 등에서 포스트 검색이 원활하도록 도와주는 기능을 한다.
  • Post Layout: 포스트 위, 아래에 구글 애드센스, 광고, 배너 이미지 등을 넣을 수 있다. 전체 포스트에 일괄 적용 할 수도, 단일 포스트에만 적용 할 수도 있다. 현재는 삭제했지만, 구글 애드센스와 알라딘 TTB 광고를 넣는데 사용했었다.
  • External Links: 포스트 내 외부 하이퍼링크에 파란색 밑줄표시 외에 추가로 아이콘을 표현해준다. 위키피디아의 형식을 빌려온 것으로 가독성을 향상시킨다.
  • Facebook Comments: 포스트 댓글을 페이스북 계정 연동을 통해 입력할 수 있게 해준다. 속도 때문에 많은 소셜미디어 플러그인을 사용하지 않고 있는데, 유일하게 유지하는 플러그인.
  • Akismet: 스팸댓글을 필터링 해준다. 가장 파워풀한 서비스로 유명. Wordpress 사용 중에 스팸댓글 하나도 없었다. 거의 완벽.
  • W3 Total Cache: 블로그 캐시 삭제를 통해 속도를 올려준다. 접속시 30초 hang 걸린 문제가 있었는데, 이 기능으로 캐시를 정리해서 문제가 해결됐다.
  • Faster Image Insert: Wordpress 이미지 업로드 기능의 불편한 것 중 하나가 여러 이미지를 업로드 하는 것은 되지만, 포스트에 한 번에 넣는 것이 안되는 것인데, 그 부분을 해결해주는 플러그인이다.
  • Jetpack by Wordpress.com: 사용자 통계 플러그인으로 Wordpress.com의 공식 플러그인. 매일 방문자 순, 포스트 별 뷰수, 리퍼러(어디를 통해 들어오는지)를 확인 할 수 있다. 필수 플러그인 중 하나.
  • Wassup: Jetpack은 하루의 방문자 통계에 강하다면, Wassup은 실시간 방문자 확인, 방문자의 IP, 도메인 상세정보를 확인하는데 강력한 기능을 갖고있다.
  • Worepress Popular Posts: 블로그 포스트 중 방문자 통계를 통해서 인기 있는 포스트 순으로 정리해주는 기능.

 

테마

Wordpress 테마의 종류는 오픈소스로 무료로 쓸 수 있는 테마, 프리미엄으로 불리는 유료 테마, 그리고 직접 테마를 만들어 쓰는 경우로 구분 할 수 있다.

현재 블로그에서 사용하는 테마는 TRIPLESHIPS라는 디자인 회사에서 과거에 오픈소스로 배포(현재는 공식 배포 중단)한 tanzaku 테마의 스타일 수정을 통해 색상, 그리드 사이즈 등을 변경해서 사용하고 있다(Github 링크).

 

 

설치형 블로그는 설정, 관리, 운영에 에너지를 어느정도 쏟아야 하는 것은 사실이지만 CSS나 Javascript, PHP에 관심이 있거나 테마 개인화, 플러그인 설정 등에 높은 자유도를 원하는 사용자라면 서비스형이 아닌 설치형 블로그를 경험해보는 것도 추천한다.

더불어 Wordpress 설치형을 사용하고, 문제가 생겼을 때 개발자 포럼을 통해 질문/답변도 해보고, 여러 테마/플러그인 개발자들의 모습들을 보면서 오픈소스와 오픈소스를 통한 가치공유, 오픈소스를 통한 수익창출에 대한 메커니즘을 피부로 경험했다. Proudly Powered by Wordpress!
